# LogiTrack PWA

Mobile inventory tracking тАФ scan-first, offline-first. Direction A ┬╖ Industrial Blue (dark).

## Stack

- Vite 6 + React 18 + TypeScript (strict)
- React Router v7 (`createBrowserRouter` + `RouterProvider`)
- Zustand (ui/auth state) + TanStack Query (server cache)
- Dexie 4 (IndexedDB) + `dexie-react-hooks`
- `vite-plugin-pwa` (injectManifest) + Workbox 7 (precache + Background Sync queue)
- Native `BarcodeDetector` тЖТ `@zxing/browser` wasm fallback for QR/EAN
- Zod for runtime validation at system boundaries

## Dev

```bash
pnpm install        # or npm / bun
pnpm dev            # Vite + SW (enabled in dev via devOptions)
pnpm typecheck
pnpm lint
pnpm build          # emits dist/ with precached shell
pnpm preview
```

Then open `http://localhost:5173` on your phone (same LAN). For real camera and Background Sync you need **HTTPS**; use `ngrok http 5173` or `vite-plugin-basic-ssl`.

## Offline-first write path

1. UI calls `txRepo.create(tx)`.
2. Dexie transaction writes `transactions` + `items.stock` adjust + `syncQueue` enqueue тАФ atomic.
3. App UI reflects change via `useLiveQuery`.
4. When online, SW intercepts `POST /api/tx` and queues via Workbox `BackgroundSyncPlugin` (`tx-upload-queue`).
5. On `sync` event the browser replays queued requests. On 2xx, write-back `txId` + `syncedAt`.

## PWA install

- Manifest: `vite.config.ts` (name, colors, orientation).
- Icons: replace `public/icons/icon-{192,512,maskable-512}.png` with real assets (not generated by scaffold).
- iOS Safari supports install but not Background Sync тАФ rely on app-open triggered drain there.

## Backend (MongoDB Atlas + Vercel Functions)

CRUD API lives under `api/` тАФ Vercel deploys each file as a serverless function.

### Endpoints
- `GET  /api/items` тАФ list all
- `POST /api/items` тАФ create (body: `{sku,name,loc,zone?,ean?,stock?,reorderAt?,unit?}`)
- `GET  /api/items/:sku` тАФ read one
- `PATCH/PUT /api/items/:sku` тАФ update
- `DELETE /api/items/:sku` тАФ delete
- `GET  /api/tx` тАФ list transactions
- `POST /api/tx` тАФ create (idempotent on `localId`; adjusts item stock)

### Setup
1. Create a free MongoDB Atlas cluster.
2. Atlas тЖТ Network Access тЖТ allow `0.0.0.0/0` (Vercel functions use dynamic egress IPs).
3. Atlas тЖТ Database Access тЖТ create a user with readWrite on the `logitrack` DB.
4. Copy the SRV connection string.
5. Locally: copy `.env.example` тЖТ `.env.local`, paste URI.
6. Vercel: `vercel env add MONGODB_URI production` then `vercel env add MONGODB_DB production`.
7. Redeploy.

### Client flow
- Boot: `hydrateFromServer()` pulls `/api/items` into Dexie (falls back silent if offline).
- Add/edit/delete: `/inv/new`, `/inv/:sku/edit` call `api.items.*` then mirror to Dexie.
- Transactions: `txRepo.create` writes Dexie first, then `POST /api/tx`; Workbox Background Sync replays failed writes when connectivity returns.

## Scope caveats (v1)

- **OCR screen** is a stub тАФ wiring `tesseract.js` or ML Kit (WASM) is a v2 task.
- **Auth** is local-only тАФ no backend. `/api` endpoints are placeholders; SW queues them anyway to exercise the offline path.
- **e2e tests** not set up. Vitest is wired for unit.
- **Update prompt** force-reloads on new SW (simple). Swap for a toast later.
